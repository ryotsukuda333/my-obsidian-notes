# アクセンチュア在籍中に備えるWeb系自社開発企業への転職戦略

## ✅ 技術コミュニティ・イベントでのアンテナ活動
- Connpass, Doorkeeper, Meetup → 勉強会や登壇機会を探す
- 主要カンファレンス例：
  - PyCon JP（Python）
  - JSConf JP（JavaScript）
  - AWS Summit（インフラ）
- Slack型コミュニティ（DevLOVE、エンジニア井戸端会議など）に参加

## ✅ スカウト型サービス活用（受け身でチャンスを得る）
- BizReach：年収800万円以上がターゲット。審査あり
- Findy：GitHubスコアでスカウト。偏差値化で定量的に実力を可視化
- Wantedly：カジュアル面談ベース。スタートアップ多め
- paiza：スキルチェックでスカウト精度UP
- Green：Web系特化の求人。メガベンチャー多数

### スカウトを増やすポイント
- プロフィールに具体的な実績を書く（技術・役割・成果）
- GitHubリンクや技術記事URLを記載
- スキルタグを網羅して記入（例：React, TypeScript, AWS）

## ✅ 自社開発企業が見るスキル・経験
- モダン技術の業務経験（例：TypeScript, Next.js, Kubernetesなど）
- チーム開発経験（Git, CI/CD, Pull Request運用）
- ビジネス視点（なぜそれを作るかの理解）
- アウトプット力（設計書、ドキュメント、レビュー文化）

## ✅ ポートフォリオの作り方
- GitHub：個人開発プロダクトを1つ以上公開
- 技術記事：Zenn or Qiitaで習得知識や開発ノウハウを公開
- READMEに概要・技術スタック・デモURL・工夫点を書く
- 転職用に「成果物まとめ」リポジトリを作るのも有効

## ✅ Tech Lead的経験の積み方（肩書きがなくてもOK）
- 技術選定や設計方針を主導する
- チームメンバーの育成・レビューを担当する
- タスク管理や進捗調整を能動的に行う
- プロジェクト全体の技術的方向性を支える動きを記録・言語化

## ✅ SNSや技術プラットフォームでの発信戦略
- **Twitter（X）**：業務での気づきやTipsを日常的に発信。エンジニアとの交流・発見が多い
- **Zenn**：技術記事＋Scrapで気軽にアウトプットできる。RSS登録されやすい
- **Qiita**：ノウハウまとめや開発Tipsでアクセスを稼ぎやすい
- **YouTube/Podcast**（余力があれば）：登壇動画や技術語りも有効

## ✅ アクセンチュア出身者の強みと注意点
### 強み：
- 大規模プロジェクト経験
- 顧客折衝／ドキュメント能力
- 業務理解力・課題整理力

### 注意点：
- 「実装してないでしょ？」と思われる
  - → コードを書いた証拠（GitHub, PR数など）を用意
- 貢献範囲が曖昧になりがち
  - → 自分が設計・実装・提案した具体例を職務経歴で示す
- ビジネスワード多用はNG
  - → 面接ではプロダクト愛・技術志向を強調する

## 📌 補足
- 転職準備は「外部との接点を作ること」から始める
- 技術以外の「自走力・成長意欲・発信力」が評価される
- エージェントを使わなくても、スカウトとアウトプットで十分戦える
